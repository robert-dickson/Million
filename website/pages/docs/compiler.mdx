# Optimizing Compiler

> "Compilers \[...\] are like 95% about convenience features, actually, and 5% about performance" [- Fabio Spampinato (author of Voby)](https://dev.to/fabiospampinato/voby-simplifications-over-solid-no-babel-no-compiler-5epg)

Million.js has an optimizing compiler that can be used to optimize block initialization. In accordance with Fabio's quote, it also is planned to support other features like React Hooks, non-primitives, and auto-optimization of components in Next.js.

<Callout>
  This is an experimental feature. It is not recommended to use this in
  production. Furthermore, it is currently intended to be used with the internal API.
</Callout>

It current supports [Vite](https://vitejs.dev), [Webpack](https://webpack.js.org), [Rollup](https://rollupjs.org), [RSPack](https://www.rspack.dev/), and [ESBuild](https://esbuild.github.io/) through [`unplugin`](https://github.com/unjs/unplugin).

# Experimental Compiler

> "Compilers \[...\] are like 95% about convenience features, actually, and 5% about performance \[for web environments\]" [- Fabio Spampinato (author of Voby)](https://dev.to/fabiospampinato/voby-simplifications-over-solid-no-babel-no-compiler-5epg)

Million.js has an optimizing compiler that can be used to optimize block initialization in `optimize` mode. The compiler supports other features like React Hooks, non-primitives, and auto-optimization of components in `react` mode.

It current supports [Next.js (beta)](https://nextjs.org), [Vite](https://vitejs.dev), [Webpack](https://webpack.js.org), [Rollup](https://rollupjs.org), [RSPack](https://www.rspack.dev/), and [ESBuild](https://esbuild.github.io/) through [`unplugin`](https://github.com/unjs/unplugin).
